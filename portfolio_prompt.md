# 个人作品集网站 — 完整开发 Prompt

## 任务概述

基于现有个人作品集网站做全站结构改版。生成完整可运行的 `index.html`（HTML + Tailwind CSS CDN + 原生 JavaScript，无任何框架/后端依赖，单文件即可运行）。

---

## 一、整体页面结构

页面纵向由上到下分为以下部分，**无任何点击跳转、无子页面**，所有内容全部平铺在首页：

1. **顶部导航栏**（固定悬浮）
2. **Hero 区**（极简，仅两行文字）
3. **模块1：Ae 作品展示**
4. **模块2：AIGC 作品展示**
5. **模块3：Ae + AIGC 协同作品**
6. **个人介绍与联系方式**
7. **页脚**（含隐藏入口）

---

## 二、导航栏规范

- 固定在页面顶部，始终可见
- 左侧：作品集名字「张峻烨」（小字，低调）
- 右侧：浮动按钮组（圆形或方形小图标）
  - **深色/浅色切换按钮**（图标式，月亮/太阳）
  - 无其他可见按钮
- 深浅切换按钮悬浮在页面右上角，固定定位，不随滚动消失

---

## 三、Hero 区规范

- 极简两行，无任何额外信息
- 第一行小字：「效果创意设计 × AIGC 创作」（灰色，字号小）
- 第二行大字：「张峻烨」（主标题，大字号）
- 不需要副标题、标签、按钮、描述

---

## 四、三大作品模块通用规范

### 4.1 模块标题区
每个模块顶部有：
- 分隔线
- 模块大标题（编辑模式下可直接点击修改）
- 模块简介（一句话，编辑模式下可修改）

### 4.2 背景色块区分（由浅到深）
- 模块1（Ae 作品）：最浅的背景色
- 模块2（AIGC 作品）：中间色
- 模块3（Ae + AIGC 协同）：最深的背景色
- 深色模式与浅色模式下各自维护一套深浅梯度

### 4.3 每行卡片数量控制
- 每个模块**独立设置**每行显示数量
- 编辑模式下，每个模块标题旁显示一个**数字输入框**（如「每行: [4]」），可直接输入数字修改该模块的每行卡片数量
- 默认值：mp4/图片模块默认 4，webp 模块默认 5

### 4.4 编辑模式下的卡片操作
- 鼠标悬停卡片时，显示「上传」和「删除」两个操作按钮
- 「删除」按钮点击后直接移除该卡片（无需确认）
- 每个模块底部有「+ 添加作品」按钮（仅编辑模式可见）
- 添加新卡片后自动触发上传文件对话框

## 五、模块1：Ae 作品展示

### 媒体类型
- **仅支持 mp4 视频**（上传时自动识别文件格式）
- 上传其他格式自动提示「请上传 mp4 或 webp 文件」

### 布局
- 瀑布流 / Masonry 布局
- 卡片宽高比**自适应视频原生宽高比**（16:9 就 16:9，9:16 就 9:16）
- 不强制统一高度，不裁剪画面

### mp4 卡片规范
- 封面：视频上传后**自动截取第一帧**作为封面图（JS Canvas 实现）
- 封面上显示播放图标（居中）
- 不自动播放
- **点击卡片后：原地展开变大播放**
  - 展开尺寸：屏幕宽度的 60%
  - 展开后浮在原位置上方，周围其他卡片不移动（用绝对定位实现）
  - 播放器使用浏览器原生 `<video controls>`（含进度条、暂停、音量、全屏）
  - 再次点击卡片或点击展开区域外部，收起关闭
- 卡片下方信息区：
  - 标题（编辑模式可点击修改，展示模式锁定）
  - 描述（多行无上限，编辑模式可修改，展示模式锁定）

### 子分类（仅用于本地管理，页面上不做区分）
- 完整作品类（MG动画、动漫剪辑、影视剪辑、文字动画、木偶动画）
- 非完整作品类（人物特效、文字动画、特效滤镜、画面特效、视频模版、转场）
- **页面上全部混排，不显示子分类标题**

## 六、模块2：AIGC 作品展示

该模块内部分为**三个子区域**，用小标题 + 分隔线区分，但同属一个模块背景色块内：

### 6.1 子区域一：图片类

- 上传格式：jpg / png / webp（静态图）
- 布局：规整网格（非瀑布流，统一宽高比，推荐 4:3 或 1:1）
- 卡片：仅图片，描述文字**可选**
  - 有描述：图片下方显示文字
  - 无描述：只显示图片，无占位框
- 每行数量独立控制

### 6.2 子区域二：视频类

- 规范与模块1 的 mp4 卡片完全相同：
  - 自动截取第一帧封面
  - 点击原地展开 60% 播放
  - 浏览器原生控制栏
  - 标题 + 可选描述

### 6.3 子区域三：Prompt 类

- **纯文字卡片，无图片**
- 卡片样式与其他卡片背景一致（卡片背景色）
- 每张卡片包含：
  - 标题（粗体，较大字号）
  - 正文（Prompt 内容，等宽字体或普通字体皆可）
- **折叠逻辑**：
  - 默认显示：标题 + 正文前 3 行
  - 超出 3 行时显示「展开全文」按钮
  - 点击展开后显示完整内容，按钮变为「收起」
- 编辑模式下标题和正文均可点击直接修改

---

## 七、模块3：Ae + AIGC 协同作品

- 支持 mp4 上传（与模块1 规范一致）
- 支持 webp / gif 上传（与下方 webp 规范一致）
- 上传时自动识别格式：
  - `.mp4` → mp4 卡片逻辑
  - `.webp` / `.gif` → webp 卡片逻辑
  - `.jpg` / `.png` → 静态图片卡片逻辑
- 卡片规范按文件格式自动匹配对应展示方式

---

## 八、WebP / GIF 动图卡片规范（适用于含动图的模块）

- 高密度小窗口网格，多卡片铺满，不够一行就排到下一行
- 卡片尺寸**统一小尺寸**，规整网格（非瀑布流）
- 所有 webp/gif：
  - 用 `<img>` 标签直接展示，天然循环播放
  - **全程自动播放，加载完即持续循环**
  - **无任何鼠标悬停交互**（不放大、不停止、不 overlay）
  - **无任何点击交互**（点击无反应）
  - 取消一切 hover 效果
- 卡片下方可选标题（编辑模式下可修改）

## 十、编辑模式系统

### 10.1 入口（隐蔽设计）
- **页脚角落**放一个极小的图标（如齿轮⚙或钥匙🔑），颜色与背景接近，低调
- 点击后进入编辑模式（无需密码）
- GitHub 配置面板入口也在同一位置（与编辑入口合并，点击后弹出面板含「进入编辑」和「GitHub 配置」两个选项）

### 10.2 编辑模式下的可见元素
- 每个模块标题旁显示「每行: [N]」数字输入框
- 每张卡片悬停时显示「上传」和「×删除」按钮
- 所有文字内容（标题/描述/模块简介/个人介绍）变为 `contenteditable`，显示虚线边框提示可编辑
- 每个模块底部显示「+ 添加作品」按钮
- Prompt 卡片文字可直接编辑

### 10.3 展示模式
- 所有编辑入口、上传按钮、删除按钮全部隐藏
- 内容锁定，不可编辑
- 纯浏览状态

## 十一、GitHub 存储系统

- 所有上传的文件通过 GitHub API 推送到用户指定的 GitHub 仓库
- 生成永久 raw 链接：`https://raw.githubusercontent.com/{user}/{repo}/main/media/...`
- 上传流程：选择文件 → 本地 blob URL 即时预览 → 后台上传 GitHub → 上传完成后替换为永久链接
- 配置信息（用户名/仓库名/Token）保存在 localStorage
- GitHub 配置面板包含：
  - 用户名、仓库名、Token 三个输入框
  - 「测试连接」按钮
  - 「保存」按钮
  - 操作说明文字

## 十二、视觉风格规范

### 12.1 深色模式（默认）
- 背景色：轻微带黄棕色调的暖灰，如 `#1a1816`（最底层）
- 模块背景梯度（由浅到深）：
  - 模块1：`#1f1d1a`
  - 模块2：`#1c1a17`
  - 模块3：`#191714`
- 卡片背景：比模块背景明显更亮，如 `#2a2724`
- 主文字色：`#e8e4dc`（暖白）
- 次级文字色：`#7a7570`（暖灰）
- 第三级文字色：`#4a4540`
- 分隔线：`rgba(255,255,255,0.06)`
- 强调色：**无**（纯黑白灰暖色系，不使用任何彩色高亮）
- 链接 hover：文字变亮

### 12.2 浅色模式
- 背景色：`#f5f4f2`（暖白）
- 模块背景梯度（由浅到深）：
  - 模块1：`#f8f7f5`
  - 模块2：`#f2f1ee`
  - 模块3：`#edece9`
- 卡片背景：`#ffffff`
- 主文字色：`#1a1816`
- 次级文字色：`#666360`
- 第三级文字色：`#999690`
- 分隔线：`rgba(0,0,0,0.07)`
- 强调色：**无**（纯黑白灰暖色系）
- 参考：modelscope.cn/home 的整体页面密度感

### 12.3 字体
- 正文：Noto Sans SC（300/400/500）
- 标签/代码/小字注释：Space Mono
- 大标题：可用 Noto Sans SC 700 加粗，无需额外装饰字体
- 字号整体克制，不使用超大字号

### 12.4 卡片设计
- 圆角：4px（统一）
- 阴影：轻微，深色模式下几乎无阴影，浅色模式下有细微投影
- hover 效果：轻微边框描边变亮（不放大、不位移）
- 过渡动画：所有 transition 0.15s-0.2s，不要夸张的弹跳/缩放

## 十三、技术规范

- 纯 HTML + Tailwind CSS CDN + 原生 JavaScript
- 不使用任何框架（React/Vue/Next 等）
- 不使用任何第三方 JS 库（GSAP/Swiper/jQuery 等）
- Tailwind 可使用 CDN 引入
- 单文件 `index.html` 即可运行，打开即可预览
- 视频第一帧截取：用 `<canvas>` + `video.currentTime = 0` 实现
- webp 循环播放：直接用 `<img src="...webp">` 标签，天然支持
- mp4 原地展开：用 CSS transform + position: fixed/absolute + z-index 实现，不用 modal 遮罩
- 每行数量控制：CSS `grid-template-columns: repeat(N, 1fr)` 动态修改
- 所有文字修改：`contenteditable="true"`
- 数据持久化：修改内容保存到 `localStorage`（JSON 序列化整个数据结构）
- GitHub API 上传：使用 `PUT /repos/{owner}/{repo}/contents/{path}` 接口
